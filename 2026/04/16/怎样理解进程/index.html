怎样理解进程

基本概念

操作系统必须全方位地管理计算机系统中正在运行的程序，因此操作系统为正在运行的程序建立了一个管理实体——进程。

  • 进程是一个具有独立功能的程序关于某个数据集合的一次运行活动

    时间，数据集，代码：三者有一个不同就不是同一个进程，而是两个不同的进程。

  • 进程是操作系统关于资源分配与处理器调度的一个独立单位

  • 进程的组成：

    1. 操作系统管理运行程序的数据结构P；
    2. 运行程序的主存或虚拟内存代码C；
    3. 运行程序的主存或虚拟内存数据D；
    4. 运行程序的通用寄存器信息R；
    5. 操作系统控制执行的程序状态字PSW；
关键理解
从原理上看，进程是支持程序执行的一种系统机制，它对处理器上运行程序的活动进行抽象； 从实现角度看，进程是一种数据结构，用来准确刻画程序的状态和系统的动态变化状况；

进程作用

刻画运行程序之间的并发性

  • 什么是程序的并发性？

    多个程序在同一时间间隔内交替推进、共同向前执行的现象。

    具体理解

    • 从宏观上看，多个程序似乎是在同时运行。
    • 但在单核 CPU 上，实际上通常是通过时间片轮转等方式，让多个程序快速交替执行，形成并发。
    • 在多核 CPU 上，部分程序也可能真的同时执行，但“并发”更强调的是多个任务在同一时间段内共同推进，不要求必须真正同一时刻都在运行。
    • 在程序设计的角度来看，用户可以设计若干个交互的可运行程序，通过共享程序共同完成一个计算问题的求解过程。

    🌰

    比如你同时：

    • 一边听音乐
    • 一边写文档
    • 一边下载文件

    这些任务看起来是一起进行的，这就是并发。

    和“并行”的区别？

    • 并发：多个任务在同一时间段内交替或部分重叠执行。
    • 并行：多个任务在同一时刻真正同时执行。

  • 由上面我们可以看出，运行程序的执行可能不是连续完成的，而是走走停停的

  • 程序自身这个概念，只是对于计算任务的指令和数据的描述，这种静态概念无法刻画程序的并发性，需要寻找可以描述程序并发执行过程的概念，即进程。

解决资源的共享性

  • 一个重要概念——可再入程序

    能够被多个程序调用的程序，本质纯代码，执行的时候不能被修改，哪个应用程序调用它哪个应用程序提供数据工作区。

    🌰

    编译器程序，编辑器程序等

  • 可再入程序的概念是一种便利化的设计，也在客观上要求了程序本身与程序的执行不再对应，而是程序与各用户程序自有的数据工作区结合构成了不同的进程

进程的状态

我们采用从特殊到一般，从局部到整体的一种理解方式

  • 进程在执行过程中需要定义三种状态（执行过程里任何时刻都在其中一种状态）

    • 运行态(Running) : 进程占有处理器正在运行的状态；

      一个显而易见的道理是，运行态的进程个数不能大于处理器的个数。

    • 就绪态(ready) : 进程具备运行条件，等待系统分配处理器以便运行的状态；

    • 等待态(waiting) : 又称阻塞态(blocked)/睡眠态(sleep) ，不具备运行条件，等待某个事件完成的状态；

  • 通常进程在创建之后处于就绪态

  • 执行中的状态转换：

    • 运行态󠀩➡等待态：出现等待事件
    • 等待态➡就绪态：等待事件结束
    • 就绪态➡运行态：系统选中，分配处理器
    • 运行态➡就绪态：被抢占或者时间片到

  • 如果进程不处于执行过程中？——执行之外进程的状态

    除了执行中，进程还存在新建态和终止态

    • 新建态(new) : 进程被创建时的状态，此时进程还没有进入就绪队列；

      创建进程
      步骤一：为进程分配所需资源以及建立必要的管理信息；
      步骤二：设置为就绪态并等待被调度执行；

    • 终止态(exit) : 有三种情况，处于终止态的进程不可再被调度执行

      • 进程完成任务达到正常结束点；
      • 出现无法克服的问题异常终止；
      • 被操作系统或者有终止权限的进程所终止；
      终止进程
      步骤一：等操作系统或者相应进程进行后续处理；
      步骤二：回收占用的资源并被系统删除；

  • 如果进程不在内存里？——内存之外的进程的状态

    实际可能出现的场景：

    1. 不断创建进程，主存资源已经不足以满足进程运行的要求，可能出现性能低或者死锁的问题，必须要将某些进程挂起并且对换到磁盘对换区里释放它占有的资源，暂时不参与低级调度；
    2. 系统出现故障，需要暂时挂起一些进程，故障消除之后再解除挂起，恢复进程运行；
    3. 用户调试程序过程可以请求挂起这个进程来进行检查修改；

    因此引入两个新状态：挂起就绪态(ready suspend)和挂起等待态(blocked suspend)，挂起状态不参与处理器调度直到他们被对换进主存；

    • 挂起就绪态(ready suspend) : 具备运行条件，但是目前在如磁盘等外存里；
    • 挂起等待态(blocked suspend) : 正在等待事件发生且在外存；

进程的数据

进程控制块（PCB）

PCB = process control block

  • 作用

    管理进程，是进程存在的唯一标识，记录刻画进程状态和环境信息，是操作系统掌握进程的唯一资源结构和管理进程的主要依据；

  • 组成

    • 标识信息：

      • 用于唯一地标识一个进程。
      • 组成：
        • 系统分配的标识号；
        • 系统分配的进程组标识号；
        • 用户定义的进程名；
        • 用户定义的进程组名；

    • 现场信息

      • 存放该进程运行时的处理器现场信息

      • 组成

        • 用户可见寄存器内容：数据寄存器，地址寄存器

        • 控制与状态寄存器内容：PC，IR，PSW

          PCB 和 PSW 的区别

          一句话记忆： PCB 是“进程的管理档案”，PSW 是“CPU 的运行状态”。

          PCB（进程控制块）

          • 操作系统为 每个进程 建立的数据结构
          • 保存进程的管理信息，例如：
            • PID
            • 进程状态
            • 程序计数器 PC
            • CPU 寄存器内容
            • 优先级
            • 打开的文件
            • 内存管理信息

          理解： PCB 更像进程的“档案袋”。

          PSW（程序状态字）

          • CPU 中保存的状态寄存器
          • 记录当前处理器运行状态，例如：
            • 条件码
            • 中断屏蔽位
            • 运行模式
            • 特权级信息
            • 下一条指令相关信息

          理解： PSW 更像 CPU 当前执行现场的一部分。

          区别

          • 位置不同：
            • PCB 在 内存中
            • PSW 在 CPU 内部
          • 作用不同：
            • PCB 用于 管理进程
            • PSW 用于 描述 CPU 的运行状态
          • 保存内容不同：
            • PCB 保存的信息更全面
            • PSW 主要保存处理器状态信息
          • 切换时的关系：
            • 进程切换时，CPU 会把当前运行状态保存到 PCB
            • 其中 PSW 是现场信息的一部分

          记忆口诀：
          PCB 管进程，PSW 管 CPU 状态

        • 栈指针：核心栈与用户栈指针

      • 进程让出处理器时，必须要将此时的现场信息保存在进程控制块中，以便进程恢复时恢复处理器现场；

    • 控制信息

      • 用于存放与管理、调度进程相关的信息

      • 组成较多，见图示

  • 进程控制块包含管理一个进程需要的全部信息，实际上定义了一个进程的当前状态，使用权和修改权属于操作系统。

  • 进程控制块在进程创建时创建，进程被撤销时回收其进程控制块所占有的空间。
