词法分析-正则表达式

本文最后更新于 2026-03-27T11:01:00+08:00

编译原理第一节：词法分析

定义

词法分析：读入源程序的输入字符，将他们组成词素，生成并输出一个词法单元序列，每个词法单元对应于一个词素。

  • 如何实现：由语法分析器调用，需要时不断读取，生成词法单元。（作用：避免额外输入输出！）

  • 其他操作：一些简单处理，删除注释，连续空白符的压缩等。

  • For example(经典例子):

    position = initial + rate * 60

    分析：它的词法单元序列：<id,1> <=,> <id,2> <+,> <id,3> <*,> <number,4>

各种概念

词素：Lexeme,源程序中的字符序列，和某类词法单元的模式匹配，被词法分析器识别为该词法单元的实例，是源程序中具有某种词法含义的一个字符序列，例如标识符“sum”、某个特定的关键字“if”、某个关系算符“>”等。

词法单元：Token，单元名（Token-name，某种词法单位抽象符号，可确定词法单元序列的结构） + 可选属性值(attribute-value)。

关于词法单元的解释：

  • 词素可以表示一类符号，比如整数，操作符，标识符，如何描述？模式匹配！
  • 一个模式匹配多个词素；
  • 属性负责传递附加的信息；
  • 属性值用于语义分析，代码生成阶段；
  • 不同的目的需要不同的属性；
  • 属性值应该是一个结构化数据

模式怎么表示：正则表达式。

正则表达式

基本概念

  • 字母表：一个有限的符号集合。

    eg. ASCII，Unicode（包括字母，数位和标点符号）

    二进制符号表：{0，1}

  • 串：字母表中符号组成的一个有穷序列；

    • 空串ε，长度为0
    • 串s长度为|s|

  • 语言：给定字母表上一个任意的可数的串的集合

    • 合法c语言的集合
    • 汉语，英语等

  • 前缀：从串的尾部删除0个或多个符号之后得到的串

  • 后缀：从串的开始删除0个或多个符号之后得到的串

  • 子串：删除某个前缀和某个后缀后得到的串

  • 真前缀，真后缀，真子串：符合以上相应定义且不等于ε或原串

  • 子序列：原串删除0个或多个符号后得到的串

串的运算：

  • 串的连接：
    • x = dog， y = house, xy = doghouse
  • 指数运算（幂运算）：
    • x = dog, x0 = ε, x2 = dogdog, x3 = dogdogdog …

语言上的运算

  • L和M的并 L∪M = {s|s属于L或者s属于M}
  • L和M的连接 LM = {st|s属于L且t属于M}
  • L的Kleene闭包 L* = ∪i = 0∞ Li ;
  • L的正闭包 L+ = ∪i = 1∞ Li

每个正则表达式r都可以描述一个语言L (r) ,即其定义的正则集合。

正则表达式定义

正则表达式可以由较小的正则表达式递归构建

  • ε是一个正则表达式，L(ε) = {ε}

  • 如果a是∑上的一个符号，a是正则表达式，L(a) = {a}

  • 选择：|

  • 连接：rs

  • 闭包：*

  • 括号：（）

  • 优先级：* > 连接 > |

  • 举例：∑ = {a, b}

    L(a|b) = {a, b}

    L((a|b)(a|b)) = {aa, bb, ab, ba}

    L(a*) = {ε,a,aa,aaa}

    L((a|b)*) = {ε, a, b, aa, ab, ba, bb, aaa…}

  • 正则表达式的等价：

    两个正则表达式r和s表示同样的语言，则r = s

正则定义

  • 对正则表达式命名：

    d1 —> r1

    d2 —> r2

    …

    dn —> rn

    • 注：di都不在字母表里，但是结合了字母表和前i-1个di的定义（可以调用之前的d）。
    • 其实就是一个简单的递归表示法，看起来更加有条理性。

  • 例子：Pascal无符号数集合

正则表达式的拓展

  • 一个或多个：r+
  • 零个或一个：r?
  • 字符：[abc] = a|b|c, [a-z] = a|b|…|z
