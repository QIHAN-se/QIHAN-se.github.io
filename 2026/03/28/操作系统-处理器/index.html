操作系统-处理器

本文最后更新于 2026-03-28T14:17:49+08:00

处理器

中央处理器结构

  • CPU包括算数逻辑单元和控制单元这两个重要部分

    • 算术逻辑单元

      • 负责计算——加减乘除，整数浮点数…；
      • 好的计算机可能有乘法器或者浮点运算器；

    • 控制单元

      • 指令译码器（Instruction Decoder, ID）：负责解释指令执行；
      • 指令寄存器（Instruction Register, IR）：指令放在这里；
      • 程序计数器（Program counter, PC）：指明下一条要被执行的指令；

  • CPU可以访问储存在寄存器与主存中的数据

    • 中央处理器可以快速直接地访问各类寄存器中的数据内容；
    • 中央处理器访问主存时，需要用到两个寄存器
      • 内存地址寄存器（Memory Address Register, MAR）
      • 内存数据寄存器（Memory Data register, MDR）

  • 读数据时

    • 主存地址寄存器设置好主存地址，发出读信号，主存地址的数据就进入了内存数据寄存器。

  • 写数据时

    • 主存数据寄存器准备好数据，主存寄存器设置好目的主存地址，发出写信号，数据就从主存数据寄存器到了主存的目的地址。

  • 有多个读写数据时，可能需要多个主存地址寄存器和主存数据寄存器。

  • 和总线的关系

    • CPU内部的部件由CPU内部的内部总线连接；
    • CPU与主存都连在系统总线（地址，数据，控制）上；

  • I/O地址寄存器IOAR与I/O数据寄存器IODR

    • IOAR储存设备地址或设备端口号
    • IODR储存关键I/O参数，如特征值与传送数据的主存首地址
    • 需要I/O设备输入和输出的数据一般储存在主存数据缓冲器中
      • I/O 设备（如键盘、硬盘）比 CPU 和内存慢成千上万倍，必须通过"缓冲"来协调这个差距。
      • 减小了cpu的实现代价，并且可以实现cpu与I/O设备并行工作。

  • 高速存储器cache

寄存器

  • 用户程序可见寄存器

    • 访问存储器时间远大于访问寄存器的时间；
    • 使用寄存器可以大大提高程序处理数据的最终效率，减少访问主存次数；
    • 所有程序都可以访问，包括用户程序和系统程序
      • 通用寄存器：用来储存数据；
      • 地址寄存器：用来储存地址，分为索引，栈指针，段地址等。

  • 控制与状态寄存器

    • 记录操作系统在控制程序执行时程序的动态行为、结果等；

    • 举个🌰：程序计数器，指令寄存器；

    • 用于控制处理器的操作，被有特权的操作系统程序使用来控制程序的执行；

    • 记录内容举例：

      • 条件吗（condition code, CC），即操作系统执行完一条指令之后给这个指令一个标识，如正/零/负/溢出等结果；

    • 更多标志位：中断位，中断允许位，中断屏蔽位，处理器模式位，主存保护位等；

  • 程序状态字

    • program status word, PSW
    • 一个概念，指记录当前程序运行的动态信息；
    • 包含：上面控制与状态寄存器提到的信息；
    • 也可能是寄存器，少数计算机设置程序状态字寄存器，大部分计算机设置一组控制与状态寄存器；

指令

  • 机器指令

    • 计算机系统执行的基本命令，处理器管理的重要基本单位。

    • 作用：逻辑运算，数据传输，控制流跳转等。

    • 指令执行过程

      • 取值：处理器根据程序计数器中的地址将指令加载到指令寄存器中；
      • 解码：解译指令寄存器里的指令来决定执行的种类；
      • 执行：连接到CPU部件执行指令的运算，产生结果并且写回，在条件码里设置运算结果标志，同时程序计数器也同时自增或改变；

    • 指令的执行遵循流水线规则

  • 指令分类和权限级别

    • 计算机系统程序分为操作系统内核程序，一类是运行于操作系统环境的普通应用程序。
    • 如果让普通应用程序得到较高的权限，可能会发生混乱与系统错误；
    • 因此，指令被分为两类
      • 特权指令
        • 仅在内核态下才能使用的指令，涉及改变处理器状态，修改控制与状态寄存器内容，启动I/O设备等敏感操作。
        • 执行这些指令不仅会影响程序自身，还会干扰其他程序以及操作系统；
      • 非特权指令
        • 在内核态和用户态都可以使用的指令

用户态与内核态

  • 内核态（kernel mode），操作系统管理程序运行所处状态

    • 全部机器指令都可以在处理器使用
    • 程序可以访问任何主存单元和系统资源，并可以改变处理器状态

  • 用户态（user mode），无法执行特权指令，只能访问用户程序所在的地址空间中的资源和数据

  • 现代计算机系统为处理器建立了硬件标志位，即处理器状态位，用来定义操作系统的保护权限；

    • 0级：内核级，处理I/O操作，执行中断处理等操作；
    • 1级：系统调用级，执行文件系统调用，获得特定的和受保护的程序服务；
    • 2级：可被多个运行进程共享，允许调用库函数，读取但是不允许修改内部数据；
    • 3级：应用程序级别，收到的保护最少；

  • 一般操作系统只选用0级和3级来简单地区分了用户态和内核态；

  • 处理器状态的切换

    • 中断是处理器状态从用户态切换到内核态的唯一途经

    • 主要包含三种情况：

      • 程序运行时产生了异步中断，如I/O操作完成，运行程序被中断，转向中断处理处理程序；
      • 程序主动同步请求操作系统服务，执行系统调用，触发了系统异常，即自愿性中断；
      • 程序运行时，执行指令出现问题产生同步异常事件（如用户态执行特权指令），运行程序被打断，转向了异常程序处理程序，属于程序性中断事件；

    • 中断机制触发之后，处理器将作出中断响应，导致处理器的状态从用户态转到内核态，并且同时交换程序状态字，引出了后续的中断/异常/系统异常处理程序；

    • 怎么回去？

      • 计算机系统提供一条加载程序状态字的特权指令（比如iret），用于内核态返回用户态，之后操作系统将控制权转交给应用进程；

  • 用户栈和核心栈

    • 用户栈：
      • 操作系统在用户进程空间里开辟的一块区域，保存应用程序的子程序或函数相互调用的参数、返回值、返回点还有局部变量等；
    • 核心栈：
      • 系统栈/内核栈，主存中属于操作系统内核空间的一块区域；
      • 作用：
        • 保护中断现场，对于后面会讲到的嵌套中断，将被中断的现场信息一次压入核心栈，中断返回时逆序弹出；
        • 保护操作系统程序间相互调用的参数、返回值、返回点还有局部变量等；
    • 每个进程创建时都有一个核心栈，可读、可写、不可执行。
    • 两个栈，但是只有一个硬件栈指针，所以随着程序执行的推进和处理器状态的变化改变栈指针的取值来工作。
