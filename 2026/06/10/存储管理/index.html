一、整章总览：存储管理到底在解决什么问题

一句话理解

存储管理就是操作系统管理主存（内存）和辅存（外存）之间数据放置、访问、保护与扩充的一整套机制。

这章核心问题

操作系统中的存储管理，本质上是在回答下面几个问题：

  1. 程序放哪儿？

    • 主存空间如何分配给进程？
    • 是整块连续放，还是拆成页、段来放？

  2. 地址怎么用？

    • 程序员写的是逻辑地址/虚拟地址；
    • 机器执行需要的是物理地址；
    • 所以必须进行 地址转换（重定位）。

  3. 多个程序如何共存？

    • 多道程序环境下多个进程同时在内存中；
    • 需要共享、保护、隔离。

  4. 内存不够怎么办？

    • 利用磁盘“假装”扩充内存；
    • 形成 虚拟存储器。

二、3.1 存储管理基础

2.1 存储管理的主要模式

一句话理解

存储管理模式的核心区别在于：程序在内存中是“整块连续放”、还是“拆成页放”、还是“按逻辑段放”。

核心概念解释

1. 逻辑地址

逻辑地址又叫 相对地址，是程序员或编译后的程序在“自己看来”使用的地址。

  • 从 0 开始编号
  • 有两种形式：
    • 一维逻辑地址：例如 1200
    • 二维逻辑地址：例如 (段号, 段内地址)

2. 物理地址

物理地址又叫 绝对地址，是内存芯片中的真实地址，CPU 最终必须用物理地址访问主存。

3. 主存复用方式

课件提到两种复用思想：

按分区复用

  • 主存被划成若干分区
  • 每个程序/段占一个分区
  • 对应连续分配或分段思想

按页框复用

  • 主存被划成若干个大小相等的页框
  • 程序拆成页面后分别装入多个页框
  • 对应分页思想

四种主要模式

模式 逻辑地址形式 物理存放方式 是否要求连续 特点
单连续存储管理 一维 一个连续区 要求连续 简单，但利用率低
段式存储管理 二维 (段号, 段内地址) 多个分段区 每段连续，段间可不连续 符合程序逻辑
页式存储管理 一维 (页号, 页内偏移) 多个页框 不要求整体连续 消除外碎片
段页式存储管理 二维/三维 段内分页 段间、页间可离散 兼顾逻辑性与效率

模式关系图

graph LR
    A[程序] --> B[单连续]
    A --> C[分段]
    A --> D[分页]
    A --> E[段页式]

    B --> B1[整个程序一个连续块]
    C --> C1[按逻辑分成多个段]
    D --> D1[按固定大小分成多个页]
    E --> E1[先分段 再对每段分页]

典型例子

单连续

整个程序像一整本书，必须完整放进书架上的一大段连续空间。

分页

把书切成等厚的小册子，每册放到任意空格里，用目录记录位置。

分段

按“前言、正文、附录”划分，每部分作为逻辑单位单独管理。

段页式

先按“章节”分段，再把每章切成固定大小的小页存放。

高频考点

  • 逻辑地址与物理地址的区别
  • 一维地址空间与二维地址空间
  • 各种存储管理模式的特点与适用场景
  • 分页和分段的本质区别

易错点

  1. 分页不是按程序逻辑划分的，而是按固定大小划分的。
  2. 分段是逻辑单位，分页是物理管理单位。
  3. 逻辑地址不是物理地址，必须经过转换。
  4. 段式地址是二维；页式虽然写成“页号+偏移”，本质上通常还是一维线性空间被拆解。

简答题模板

题目：存储管理的主要模式有哪些？

答：
存储管理的主要模式包括单连续存储管理、段式存储管理、页式存储管理和段页式存储管理。
单连续存储管理要求程序占据一块连续主存空间，实现简单但利用率较低；
段式存储管理按程序逻辑结构划分为若干段，便于共享和保护；
页式存储管理按固定大小分页，消除了外碎片，便于虚拟存储实现；
段页式存储管理结合了分段和分页的优点，既保留逻辑结构，又提高存储利用率。

考前速背版

  • 逻辑地址：程序看到的地址
  • 物理地址：内存真实地址
  • 单连续：整块连续放
  • 分段：按逻辑结构分
  • 分页：按固定大小分
  • 段页式：先分段，再分页

2.2 存储管理的功能

一句话理解

存储管理的任务不只是“分内存”，还包括地址转换、分配回收、共享、保护和内存扩充。

五大核心功能

1. 地址转换（重定位）

将逻辑地址转换成物理地址。

分类

类型 发生时机 特点
静态重定位 装入内存时 一次性转换，装入后地址固定
动态重定位 程序执行时 访问时实时转换，灵活性强

为什么需要动态重定位？

因为多道程序环境中，进程装入位置不固定，运行时还可能交换、移动、分页调度。

2. 主存空间的分配与回收

分配

当进程进入内存时，OS 为其分配内存，并登记在表中。

回收

当进程结束或归还部分内存时，OS 回收相应空间并修改管理表。

3. 主存共享

共享有两层含义：

含义一：多个进程共同使用主存资源

这是多道程序设计的基本含义。

含义二：多个进程共享同一块程序或数据

例如：

  • 共享代码段
  • 共享只读库
  • 共享数据区

4. 存储保护

防止进程相互破坏。

保护原则

  • 自己的私有区：可读可写
  • 公共共享区：按授权访问
  • 别人的私有区：不可随意访问

实现方式

需要 硬件 + 软件 协同：

  • 硬件检查越界、权限
  • 出错时产生保护异常
  • OS 处理中断

5. 主存扩充

当主存不够时，把磁盘等辅存作为后备，形成“更大的可用内存”。

两种基本技术

技术 单位 特点
对换（Swapping） 整个进程 把暂不运行进程换出内存
虚拟技术 页或段 只调入所需部分，效率更高

功能关系图

graph TD
    A[存储管理功能] --> B[地址转换]
    A --> C[分配与回收]
    A --> D[共享]
    A --> E[保护]
    A --> F[扩充]

    F --> F1[对换]
    F --> F2[虚拟存储]

典型例子

假设 3 个进程同时运行：

  • P1 使用代码区、数据区
  • P2 与 P3 共用一个数学库代码页
  • P2 不能访问 P1 私有数据
  • 内存满了时，将 P3 某些页换出到磁盘

这正体现了：

  • 分配
  • 共享
  • 保护
  • 扩充
  • 地址转换

高频考点

  • 存储管理有哪些功能
  • 静态重定位与动态重定位区别
  • 为什么存储保护需要软硬件协同
  • 对换与虚拟存储器的区别

易错点

  1. “共享主存资源”不等于“共享同一块内存内容”。
  2. 静态重定位不适合需要移动的程序。
  3. 虚拟存储器不是把磁盘变成真正等速内存，它只是逻辑扩充。
  4. 保护不是只靠软件检查，必须依赖硬件。

简答题模板

题目：简述存储管理的主要功能。

答：
存储管理的主要功能包括：

  1. 地址转换，即把逻辑地址转换为物理地址；
  2. 主存空间分配与回收，为进程分配内存并在结束后回收；
  3. 主存共享，支持多个进程共享主存资源及共享代码/数据区；
  4. 存储保护，防止进程非法访问他人内存；
  5. 主存扩充，通过对换和虚拟存储技术扩大可用内存空间。
    这些功能通常需要操作系统和硬件共同配合实现。

考前速背版

  • 功能五件套：转、分、共、保、扩
  • 转：地址转换
  • 分：分配与回收
  • 共：共享
  • 保：保护
  • 扩：对换 + 虚拟存储

2.3 虚拟存储器概述

一句话理解

虚拟存储器利用“局部性原理”，让程序不必全部装入内存也能运行，从而实现“内存逻辑扩容”。

为什么提出虚拟存储器

如果没有虚拟存储器，会遇到这些问题：

  1. 程序必须完全装入内存才能运行
  2. 大程序可能因内存不足无法运行
  3. 多道程序并发数受限
  4. 程序员需要考虑内存容量，编程不方便

核心理论基础：局部性原理

课件明确强调了局部性，这是虚拟存储器成立的前提。

1. 时间局部性

如果某条指令/某个数据刚刚被访问过，那么不久后很可能再次被访问。

例如：

  • 循环变量
  • 常用局部变量
  • 热点函数

2. 空间局部性

如果某个地址被访问，那么附近地址也很可能很快被访问。

例如：

  • 顺序执行指令
  • 数组遍历
  • 连续数据结构访问

为什么局部性让“部分装入”成为可能

因为程序运行时并不会在任意时刻用到全部代码和数据，而只会集中访问其中一小部分。
于是操作系统就可以：

  • 先只装入一部分
  • 运行时需要什么再调什么
  • 暂时不用的再换出去

这就是虚拟存储的思想。

虚拟存储器的基本思想

  • 进程的全部信息放在辅存中
  • 当前运行所需部分装入主存
  • 访问未装入内容时，触发中断，由 OS 调入
  • 若主存已满，则换出暂时不用的部分

虚拟存储器示意

flowchart LR
    A[逻辑地址空间/虚拟地址空间] --> B[CPU发出虚拟地址]
    B --> C[存储管理部件/MMU]
    C --> D[若在主存中 则地址转换]
    C --> E[若不在主存中 则缺页/缺段中断]
    E --> F[OS从辅存调入]
    F --> D
    D --> G[形成物理地址]
    G --> H[访问主存]

虚拟存储器的本质

标准表述

虚拟存储器是一种 地址空间扩展技术，而不是简单的物理内存扩容。

更准确理解

它让用户感觉自己拥有比实际主存大得多的存储空间，但实际上大量内容驻留在辅存中，靠 OS 和硬件动态调度。

虚拟存储器的优点

  1. 运行比主存更大的程序
  2. 提高多道程序度
  3. 提高 CPU 利用率
  4. 提高内存利用率
  5. 对用户透明，编程方便

虚拟存储器的代价与问题

  1. 地址转换复杂
  2. 缺页会带来巨大开销
  3. 需要硬件支持
  4. 页面替换不当会产生 抖动（Thrashing）

抖动是什么

如果刚换出的页马上又要换入，系统大部分时间都在换页，而不是执行用户程序，这种现象就叫 抖动 或 颠簸。

高频考点

  • 虚拟存储器的定义
  • 虚拟存储器建立在什么基础上
  • 局部性原理的含义
  • 抖动是什么，为什么会发生

易错点

  1. 虚拟存储器不是“内存 + 磁盘 = 新内存”这么简单。
  2. 局部性不是“程序只访问一个地方”，而是“访问具有聚集性”。
  3. 抖动是频繁换页，不是简单的缺页多。
  4. 虚拟存储器的基础是局部性 + 硬件支持 + OS管理。

简答题模板

题目：什么是虚拟存储器？其理论基础是什么？

答：
虚拟存储器是操作系统利用辅存扩充主存的一种技术。它把进程的全部信息保存在辅存中，仅把当前运行所需要的一部分装入主存，在执行过程中根据需要动态调入和调出。
虚拟存储器的理论基础是 程序执行的局部性原理，包括时间局部性和空间局部性。由于程序在一段时间内只集中访问部分代码和数据，因此进程不必全部装入主存也可以正确运行。

考前速背版

  • 虚拟存储器 = 逻辑扩充内存
  • 依据 = 局部性原理
  • 机制 = 部分装入，随用随调
  • 问题 = 缺页开销、抖动

2.4 存储管理的硬件支撑

一句话理解

没有硬件支撑，现代存储管理几乎无法高效实现。

课件中的硬件支撑要点

1. 存储器层次结构

从快到慢、从贵到便宜、从小到大：

graph TD
    A[寄存器] --> B[L1 Cache]
    B --> C[L2 Cache]
    C --> D[L3 Cache]
    D --> E[主存 DRAM]
    E --> F[SSD/硬盘]
    F --> G[远程外存]

规律：

  • 越靠上：速度快、容量小、价格高
  • 越靠下：速度慢、容量大、价格低

Cache 的作用

Cache 是介于 CPU 和主存之间的高速小容量存储器。

为什么需要 Cache

CPU 很快，主存相对慢。
如果每次都直接访问主存，CPU 大量时间会等待。

Cache 为什么有效

因为程序具有局部性：

  • 最近访问过的数据很可能再次访问
  • 附近数据很可能马上访问

Cache 与存储管理的关系

课件提到，存储管理不仅管主存，还会配合 Cache、快表等机制提升性能。

注意区分：

名称 作用
Cache 缓存数据/指令块
TLB（快表） 缓存页表项，提升地址转换速度

动态重定位与保护的硬件支持

课件给出基址寄存器和限长寄存器机制。

原理

  • 基址寄存器：记录程序在内存中的起始地址
  • 限长寄存器：记录该程序可访问的地址范围长度

CPU 访问逻辑地址 LA 时：

  1. 检查 0 <= LA < 限长
  2. 若越界，触发中断
  3. 若合法，计算：
    物理地址 = 基址 + 逻辑地址

动态重定位流程图

flowchart TD
    A[CPU给出逻辑地址LA] --> B{LA < 限长?}
    B -- 否 --> C[越界中断/保护异常]
    B -- 是 --> D[物理地址 = 基址 + LA]
    D --> E[访问主存]

为什么硬件必不可少

对地址转换

每条指令可能都涉及访存，如果全靠软件转换，开销不可接受。

对存储保护

每次访问都要检查权限和边界，必须靠硬件在线完成。

对虚拟存储

缺页中断、页表机制、TLB、替换支持等都依赖硬件。

高频考点

  • 基址寄存器与限长寄存器作用
  • Cache 与主存层次结构
  • 为什么存储管理需要硬件支持
  • Cache 与 TLB 的区别

易错点

  1. Cache 不是页表。
  2. TLB 不是普通数据缓存，而是页表项缓存。
  3. 动态重定位不是装入时改地址，而是执行时转换。
  4. 限长寄存器用于越界检查，不是存基址。

题目：为什么现代存储管理必须依赖硬件支撑？

答：
现代存储管理必须依赖硬件支撑，主要原因有三点：

  1. 地址转换频繁，若仅靠软件实现，效率过低；
  2. 存储保护需要实时检查，如越界和访问权限检查，必须由硬件快速完成；

  • 层次：寄存器 → Cache → 主存 → 外存
  • 动态重定位：基址 + 限长
  • 保护：越界检查
  • 没有硬件：高效存储管理几乎无法实现

三、3.2 单连续分区存储管理

3.1 单用户连续分区与固定分区

一句话理解

连续分配的核心思想是：一个进程占用一块物理上连续的内存区域。

1. 单用户连续分区存储管理

基本思想

主存分为两部分：

  • 系统区：操作系统使用
  • 用户区：用户程序使用

系统通过 栅栏寄存器（边界寄存器）把这两个区域隔开。

特点

  • 一次只装一个用户作业
  • 通常采用静态重定位
  • 实现简单、硬件开销低
  • 适用于早期单用户单任务系统，如 DOS

2. 固定分区存储管理

基本思想

把用户区事先划分成若干个固定大小的分区，每个分区一次装一个作业。

特点

  • 分区数量固定
  • 分区大小固定（也可以不同分区大小不同，但每个分区是预先确定的）
  • 支持多道程序
  • 早期 OS 常采用

固定分区的优点

  1. 实现简单
  2. 便于管理
  3. 可支持多个作业并发驻留

固定分区的缺点

1. 内碎片

如果作业比所在分区小，多余部分浪费。

例如：

  • 分区大小 16KB
  • 作业只需要 10KB
  • 剩余 6KB 不能给别的进程使用

这就是 内碎片

2. 不灵活

大作业可能找不到足够大的分区，即使总空闲空间够也不行。

3.2 可变分区存储管理

一句话理解

可变分区按进程实际需要动态划分内存，比固定分区灵活，但会产生外碎片。

基本思想

  • 当进程创建时，按其实际需要分配一块连续内存
  • 分区大小不预先固定
  • 分区数目会随着进程装入和撤离动态变化

分配过程

Step 1：检查空闲区

查看是否存在足够大的空闲块。

Step 2：从某空闲区切分

如果有，则从中划出所需大小给进程。

Step 3：更新空闲表/分配表

记录剩余空闲区和已分配区。

Step 4：进程结束时回收

归还空间，并尝试与相邻空闲区合并。

为什么会产生外碎片

随着进程不断装入、撤离，内存中会出现很多 不连续的小空闲块。
这些小空闲块总量也许很多，但由于不连续，无法满足大进程的分配需求。

这就是 外碎片。

内碎片 vs 外碎片

项目 含义 常见于
内碎片 已分配块内部未利用的空间 固定分区、分页最后一页
外碎片 分散在已分配块之间的零散空闲区 可变分区、分段

可变分区的分配算法

这是考试重点。

1. 首次适应算法 First Fit

从低地址开始找，找到第一个足够大的空闲区就分配。

特点：

  • 简单
  • 速度较快
  • 低地址空间容易形成很多碎片

2. 循环首次适应算法 Next Fit

从上次找到的位置继续往后找。

特点：

  • 避免总从头查找
  • 平均查找速度可能更快
  • 可能使空间利用率略差

3. 最佳适应算法 Best Fit

找所有空闲区中 最小但足够大 的那一个。

特点：

  • 看起来“最省”
  • 但容易产生很多很小、很难再利用的碎片

4. 最坏适应算法 Worst Fit

找最大的空闲区来分配。

特点：

  • 每次切一块下来，剩余部分仍较大
  • 但大块资源会被快速消耗

算法对比表

算法 思想 优点 缺点
First Fit 找第一个能放的 快、简单 前部碎片多
Next Fit 从上次位置继续找 减少重复扫描 利用率可能下降
Best Fit 找最贴合的空闲区 表面节省空间 产生大量小碎片
Worst Fit 找最大的空闲区 保留较大剩余块 大块资源容易被破坏

可变分区的紧凑（压缩）

为什么需要

外碎片太多时，总空闲量足够但没有连续大块可用。

做法

把已分配进程向一端移动，合并空闲区。

代价

  • 数据移动开销大
  • 需要动态重定位支持
  • 不宜频繁做

    C -- 否 --> D[等待/紧凑/拒绝]

  2. 最佳适应不一定最好。

四、3.3 页式存储管理

4.1 页式存储管理基本原理

一句话理解

分页把程序和内存都切成固定大小的小块，用页表记录“页放在哪个页框里”。

基本概念

页（Page）

进程逻辑地址空间被划分出的固定大小块。

页框 / 页架（Frame）

主存被划分出的固定大小块。

页大小 = 页框大小

这是分页能进行映射的前提。

分页为什么出现

为了解决连续分配中的问题：

  • 可变分区有外碎片
  • 紧凑代价大
  • 连续分配灵活性差

分页通过“离散分配”解决这些问题。

分页的基本思想

  • 主存划分为多个大小相等的页框
  • 进程划分为多个同样大小的页面
  • 任意页面可装入任意空闲页框
  • 不要求整个进程连续存放
  • 页表负责维持逻辑连续性

4.2 页式存储管理的地址转换

一句话理解

分页地址转换就是：先找页表得到页框号，再和页内偏移拼成物理地址。

基本过程

Step 1：拆逻辑地址

逻辑地址拆成：

  • 页号 p
  • 页内偏移 d

Step 2：查页表

找到页号 p 对应的页框号 f

Step 3：形成物理地址

把页框号 f 和偏移 d 拼接，得到物理地址。

基本流程图

flowchart TD
    A[逻辑地址LA] --> B[分解为页号p和偏移d]
    B --> C[查页表得到页框号f]
    C --> D[物理地址PA = f×页大小 + d]
    D --> E[访问主存]

访问代价问题

如果页表在主存中，那么一次逻辑地址访问需要：

  1. 访问主存中的页表
  2. 再访问主存中的目标单元

所以通常要 访问两次主存。

这就是分页地址转换的性能问题。

快表 TLB

定义

TLB（Translation Lookaside Buffer），即 快表，是存放部分页表项的高速联想存储器。

为什么有效

程序访问地址有局部性，近期访问过的页很可能再次访问。
因此常用页表项缓存在 TLB 中，可以提高转换速度。

引入 TLB 后的流程

flowchart TD
    A[逻辑地址]
    A --> B[取页号和偏移]
    B --> C{TLB命中?}
    C -- 是 --> D[直接得到页框号]
    C -- 否 --> E[访问主存页表]
    E --> F[得到页框号]
    F --> G[更新TLB]
    D --> H[形成物理地址]
    G --> H
    H --> I[访问主存数据]

TLB 平均访问时间计算

课件给出的经典计算：

  • 主存访问时间 = 200ns
  • TLB 访问时间 = 40ns
  • TLB 命中率 = 90%

命中时

  • 查 TLB：40ns
  • 访问主存：200ns
    合计：240ns

未命中时

  • 查 TLB：40ns
  • 查页表（主存）：200ns
  • 访问数据（主存）：200ns
    合计：440ns

为什么 TLB 用联想存储器

因为 TLB 不是按“地址位置”找，而是按“页号内容”快速匹配。
这就是 按内容寻址。

高频考点

  • 为什么分页访问慢
  • 什么是 TLB
  • TLB 的作用

易错点

  1. TLB 命中时仍然要访问一次主存取真正数据。
  2. TLB 不存整张页表，只存部分热点页表项。
  3. TLB 是地址转换缓存，不是普通数据缓存。
  4. 两次主存访问：一次查页表，一次取数据。

简答题模板

题目：为什么要引入快表 TLB？

答：
在页式存储管理中，如果页表放在主存中，则每次地址转换都要先访问页表，再访问实际数据，因此通常需要两次主存访问，降低了存取速度。
为了提高地址转换速度，系统设置快表 TLB，将部分常用页表项存放在高速联想存储器中。若访问页的页表项在 TLB 中命中，就可以直接得到页框号，从而减少一次主存访问，提高系统性能。

考前速背版

  • 页表在主存：一次访存常要 2 次主存访问
  • TLB：缓存页表项
  • 命中：快
  • 未命中：查页表 + 更新 TLB

4.3 页式虚拟存储管理

一句话理解

页式虚拟存储管理就是“只把需要的页面装入内存，不在内存的页面等用到时再调入”。

基本思想

  • 进程全部页面保存在辅存
  • 运行时只装入部分页
  • CPU 访问某页时：
    • 若在内存，直接访问
    • 若不在内存，触发 缺页中断
  • OS 处理缺页，把该页调入内存

这是现代操作系统最主流的存储管理方式。

请求分页

课件提到：

首次只把进程第一页信息装入主存，称为请求页式存储管理

更标准地说：

请求分页（Demand Paging） 是指只有在访问某页时，若发现它不在主存，才将其调入。
并不是一定“只装第一页”，而是“初始只装入少量必要页，其他页按需调入”。

页表项需要增加哪些信息

普通分页只需要页框号。
虚拟存储下，页表项还要记录更多状态：

字段 作用
驻留位/存在位 该页是否在主存
页框号 若在主存，对应哪个页框
辅存地址 若不在主存，磁盘上在哪
修改位 页面是否被写过
引用位 最近是否被访问
保护位 访问权限
其他控制位 如锁定、可移动等

缺页中断处理过程

Step 1：CPU 发现页不在内存

查页表发现驻留位为 0。

Step 2：发出缺页中断

CPU 保存现场，转入 OS。

Step 3：OS 处理

  • 若有空闲页框：直接装入
  • 若没有空闲页框：先选一个页淘汰

Step 4：若淘汰页被修改

先写回磁盘。

Step 5：调入新页

从辅存读到页框。

Step 6：更新数据结构

更新页表、TLB、主存分配表。

Step 7：恢复现场，重启指令

重新执行刚才引发缺页的那条指令。

flowchart TD
    A[CPU访问逻辑地址] --> B[查TLB/页表]
    B --> C{页面在主存?}
    C -- 是 --> D[形成物理地址并访问]
    C -- 否 --> E[产生缺页中断]
    E --> F[OS保存现场]
    F --> G{有空闲页框?}
    G -- 是 --> H[直接调入页面]
    G -- 否 --> I[选择淘汰页]
    I --> J{淘汰页被修改?}
    J -- 是 --> K[写回辅存]
    J -- 否 --> L[跳过写回]
    K --> H
    L --> H
    H --> M[更新页表/TLB]
    M --> N[恢复现场]
    N --> O[重执行被中断指令]

页式虚拟存储为什么能成立

因为：

  1. 程序有局部性
  2. 不需要全部装入
  3. 页是固定大小，调度管理方便
  4. 缺页可以自动处理，对用户透明

高频考点

  • 页式虚拟存储管理的思想
  • 请求分页的含义
  • 缺页中断处理过程
  • 页表项中的状态位

易错点

  1. 缺页中断不是程序出错，而是正常机制。
  2. 缺页后处理完成，要重执行原指令。
  3. 若被换出页未修改，可不必写回磁盘。
  4. 请求分页是“按需调入”，不是“只调第一页”。

简答题模板

题目：简述页式虚拟存储管理中缺页中断的处理过程。

答：
在页式虚拟存储管理中，当 CPU 访问某逻辑地址时，如果发现该页不在主存，则产生缺页中断。操作系统接管后先保存现场，然后检查主存中是否有空闲页框：

  • 若有空闲页框，则直接从辅存把所需页面调入；
  • 若没有，则按照页面替换算法选择一个页面淘汰，若该页被修改过，还需要先写回辅存。
    随后系统把所需页面装入主存，更新页表、快表和主存分配表，最后恢复现场并重新执行引起缺页中断的指令。

考前速背版

  • 全部页在辅存，部分页在主存
  • 访问不到 → 缺页中断
  • OS 调页 + 必要时换页
  • 更新页表/TLB
  • 重执行原指令

4.4 页面调度与页面替换算法

一句话理解

当内存满了还要装入新页时，就必须选一个旧页换出去，这就是页面调度。

页面调度的定义

课件中说：

  • 当主存已满又需装入新页时
  • 必须按一定算法把某些页调出
  • 选择淘汰页的工作称为 页面调度
  • 选择淘汰页的算法称为 页面调度算法

更标准的术语常写作：页面替换算法（Page Replacement Algorithm）

缺页中断率

设：

  • 成功访问次数 = S
  • 缺页次数 = F
  • 总访问次数 = A = S + F

则：

缺页率越低，一般说明分页管理效果越好。

影响缺页率的因素

  1. 分配给进程的页框数
  2. 页面大小
  3. 程序访问局部性
  4. 页面替换算法
  5. 系统负载程度

常见页面替换算法

课件补充目录中有“页面替换算法”，虽然截图未完整展开，但这是本章常规重点，必须掌握。

1. 最佳置换算法 OPT

思想

淘汰 未来最长时间内不会再被访问 的页面。

特点

  • 理论上最优
  • 只能事后分析，实际无法精确实现
  • 常作为其他算法的比较基准

2. 先进先出 FIFO

思想

先进入内存的页先淘汰。

优点

  • 简单易实现

缺点

  • 可能把常用页淘汰掉
  • 会出现 Belady 异常：页框数增加，缺页次数反而增加

3. 最近最久未使用 LRU

思想

淘汰最近最长时间未被访问的页面。

依据

利用时间局部性：刚访问过的页，近期大概率还会访问。

优点

  • 性能较好
  • 接近 OPT

缺点

  • 精确实现代价高，需要记录访问次序

4. 时钟算法 Clock / Second Chance

思想

用循环队列和引用位近似 LRU。

过程

  • 页面组织成环
  • 指针循环扫描
  • 若引用位 = 1，则清 0，给“第二次机会”
  • 若引用位 = 0，则淘汰该页

优点

  • 实现开销低
  • 工程上常用

算法比较表

算法 核心思想 优点 缺点
OPT 淘汰未来最晚使用页 最优 无法实际精确实现
FIFO 先入先出 简单 性能差，Belady异常
LRU 淘汰最近最久未用页 效果较好 实现复杂
Clock 用引用位近似LRU 开销小，实用 不如理想LRU精确

抖动（Thrashing）

定义

页面频繁换入换出，CPU 大部分时间都在处理缺页和I/O，而不是执行程序。

原因

  • 分配页框太少
  • 并发进程太多
  • 页面替换算法不合适
  • 工作集超出可用内存

后果

系统吞吐量明显下降

页面调度逻辑图

flowchart TD
    A[需要装入新页] --> B{有空闲页框?}
    B -- 是 --> C[直接装入]
    B -- 否 --> D[执行页面替换算法]
    D --> E[选择淘汰页]
    E --> F{淘汰页已修改?}
    F -- 是 --> G[写回磁盘]
    F -- 否 --> H[直接覆盖]
    G --> I[装入新页]
    H --> I
    I --> J[更新页表/TLB]

高频考点

  • 页面调度的含义
  • 缺页率公式
  • FIFO、LRU、OPT 比较
  • 抖动的含义、原因和后果
  • Belady 异常出现在哪种算法中

易错点

  1. 页面调度本质上是页面替换。
  2. OPT 最优，但不能真正在线实现。
  3. FIFO 不考虑局部性。
  4. LRU 利用的是时间局部性。
  5. 缺页率低不一定代表绝对最好，但通常更优。

简答题模板

题目：比较 FIFO 与 LRU 页面替换算法。

答：
FIFO 算法按照页面进入主存的先后次序进行淘汰，即先进入主存的页先被换出。其优点是实现简单，但没有考虑页面的访问频度，可能把经常访问的页淘汰出去，性能较差，甚至会出现 Belady 异常。
LRU 算法淘汰最近最长时间未被访问的页面，它利用了程序的时间局部性原理，通常具有更好的性能，更接近最优置换算法 OPT。但 LRU 需要记录页面访问历史，精确实现的硬件和软件开销较大。

考前速背版

  • 页面调度 = 内存满时选页淘汰
  • 缺页率：f = F / A
  • OPT：最优，不能真实实现
  • FIFO：简单，可能 Belady
  • LRU：好用，接近最优
  • Clock：近似 LRU
  • 抖动：频繁换页

4.5 多级页表与反置页表

一句话理解

多级页表是为了解决“页表太大”；反置页表是为了让页表大小与物理内存有关，而不是与虚拟地址空间有关。

一、多级页表

为什么需要多级页表

课件给出例子：

  • 32 位逻辑地址空间：2^32 = 4GB
  • 页大小 4KB = 2^12
  • 则页面数 = 2^20
  • 若每个页表项 4B
  • 整张页表大小 = 2^20 × 2^2 = 2^22 = 4MB

一个进程就要 4MB 页表，系统中有很多进程，开销过大。

两级页表思想

把“大页表”再分页：

  • 一级页表：页目录
  • 二级页表：页表页

逻辑地址分为三部分：

两级页表地址转换过程

Step 1

根据页目录号找到对应的二级页表地址

Step 2

根据页表号找到对应页框号

Step 3

拼接页内偏移形成物理地址

两级页表图示

flowchart LR
    A[逻辑地址] --> B[页目录号]
    A --> C[页表号]
    A --> D[页内偏移]

    B --> E[查一级页表/页目录]
    E --> F[得到二级页表地址]
    C --> G[查二级页表]
    F --> G
    G --> H[得到页框号]
    H --> I[与页内偏移拼接]
    D --> I
    I --> J[物理地址]

多级页表的优点

  1. 减少连续大页表空间需求
  2. 只为实际用到的地址区间建立下级页表
  3. 适合大虚拟地址空间

缺点

  1. 地址转换更慢
  2. 访问级数增加
  3. 更依赖 TLB 加速

二、反置页表（Inverted Page Table, IPT）

为什么提出

普通页表大小与 虚拟地址空间大小 成正比。
如果虚拟空间很大、进程很多，页表非常庞大。

核心思想

不是每个虚拟页一个表项，而是每个物理页框一个表项。

也就是说：

  • 普通页表：按虚页组织
  • 反置页表：按页框组织

所以反置页表的大小取决于：

  • 物理内存页框数

而不是：

  • 虚拟页总数
  • 进程总虚页数

反置页表项包含什么

课件给出：

  • 虚拟页号
  • 进程标识符 PID
  • 各种标志位（有效、引用、修改、保护、锁定等）
  • 哈希链指针

为什么要有 PID？
因为不同进程可能有相同页号，必须用 (PID, 页号) 才能唯一标识一个虚拟页。

反置页表地址转换思想

输入

逻辑地址中的：

  • 进程标识
  • 页号
  • 位移

过程

  1. 对 (PID, 页号) 做哈希
  2. 找到哈希表入口
  3. 沿哈希链查找匹配项
  4. 若找到，其表项序号/索引就是页框号
  5. 与位移拼接形成物理地址
  6. 若找不到，产生缺页中断

反置页表图示

flowchart TD
    A[逻辑地址: PID 页号 位移] --> B[哈希函数]
    B --> C[哈希表入口]
    C --> D[遍历反置页表链]
    D --> E{找到匹配PID和页号?}
    E -- 是 --> F[表项索引即页框号]
    F --> G[拼接位移形成物理地址]
    E -- 否 --> H[缺页中断]

普通页表 vs 反置页表

项目 普通页表 反置页表
表项个数依据 虚拟页数 物理页框数
查找键 页号 (PID, 页号)
优点 直观，查找方便 节省页表空间
缺点 页表可能很大 查找复杂，常需哈希
适用 常规系统 大地址空间系统

高频考点

  • 为什么需要多级页表
  • 两级页表地址结构
  • 多级页表优缺点
  • 什么是反置页表
  • 反置页表为什么要用 PID
  • 反置页表大小与什么有关

易错点

  1. 多级页表是“页表再分页”，不是“页面多级”。
  2. 反置页表不是普通页表倒过来写，而是按页框组织。
  3. 反置页表大小取决于物理页框数，不取决于虚拟页数。
  4. 反置页表查找通常需要哈希，否则太慢。

简答题模板

题目：为什么要采用多级页表？

答：
在大虚拟地址空间系统中，普通页表会非常大。例如 32 位地址空间、4KB 页面时，一个进程的页表就可能达到数 MB，造成较大的存储开销。多级页表通过把页表再分页，将其组织成页目录和页表页两级结构，只为实际使用到的地址区间建立下级页表，从而减少页表占用的连续主存空间，提高存储利用率。其代价是地址转换过程更复杂，访问时间会增加，因此通常需要 TLB 进行加速。

考前速背版

  • 多级页表：解决 页表太大
  • 两级地址：目录号 + 页表号 + 偏移
  • 反置页表：每个页框一个表项
  • 查找键：PID + 页号
  • 优势：页表大小和物理内存有关

5.1 段式存储管理

一句话理解

分段是按程序的逻辑结构来管理内存，每段都是一个独立的逻辑单位。

分段的基本思想

程序并不总是天然适合按固定页大小切分。
从程序员角度，程序更像由以下部分组成：

  • 主程序段
  • 子程序段
  • 数据段
  • 数组段
  • 栈段
  • 工作区段

分段就是按这种逻辑结构划分。

段的特点

  1. 每段有独立意义
  2. 每段长度可变
  3. 段内地址从 0 开始
  4. 各段之间相互独立

段式逻辑地址

这是一种二维逻辑地址。

段表

系统为每个进程建立段表。
每个段表项至少包括：

  • 段基址
  • 段长
  • 访问权限/标志位

地址转换过程

设逻辑地址为 (s, d)：

  1. 检查段号 s 是否越界
  2. 查段表得到该段基址 base 和段长 len
  3. 判断 d < len 是否成立
  4. 若合法：
  5. 若 d >= len，则越界中断

分段的优点

  1. 符合程序逻辑结构

    • 用户可见，编程和理解自然

  2. 便于共享

    • 代码段、数据段可按段共享

  3. 便于保护

    • 可对不同段设置不同访问权限
    • 如代码段只执行、数据段可读写

  4. 便于动态链接、扩展

    • 某些段可扩充

分段的缺点

  1. 段长不固定，分配复杂
  2. 会产生外碎片
  3. 地址转换比单连续更复杂
  4. 需要段表管理和保护机制

5.2 段式虚拟存储管理

一句话理解

段式虚拟存储管理是把“按需调入”的思想应用到分段系统中：缺哪一段，就调哪一段。

基本思想

  • 进程所有段先存放在辅存中
  • 运行时只装入当前需要的一段或几段
  • 访问不在内存中的段时，再动态装入

这就是“按段请求调入”。

与段覆盖技术的区别

课件特意强调：

  • 段覆盖技术：用户自己安排覆盖关系，是程序设计技术
  • 段式虚拟存储：OS 自动管理，对用户透明

这是考试非常喜欢考的对比点

项目 段覆盖 段式虚拟存储
控制者 用户/程序员 操作系统
是否透明 不透明 透明
本质 手工扩充内存使用 自动虚拟化

段表扩充字段

在段式虚拟存储中，段表项要更丰富，可能包括：

  • 段号
  • 扩充位
  • 主存始址
  • 特征位（在内存/不在内存/共享）
  • 存取权限
  • 辅存始址
  • 标志位（是否修改、是否可移动等）
  • 限长

地址转换与缺段处理思路

CPU 访问 (S, B)：

  1. 检查段 S 是否在内存
  2. 若不在，发 缺段中断
  3. 若在，检查位移 B 是否越界
  4. 检查访问权限是否合法
  5. 若都合法，形成物理地址
  6. 若段可扩充且有相邻空闲区，可进行扩展处理

分段虚拟存储的特点

  1. 保留分段的逻辑结构
  2. 支持按段共享、保护
  3. 支持大程序运行
  4. 但由于段长可变，调入调出与分配更复杂

  • 缺段中断
  • 段覆盖与段式虚拟存储区别
  • 段表中新增控制信息

易错点

  1. 缺段中断和缺页中断类似，都是正常机制。
  2. 段覆盖不是 OS 自动做的。
  3. 分段虚拟存储仍然要求每个段在内存中连续。
  4. 分段虚拟存储同样可能遇到外碎片问题。

答：
二者都能在主存不足时运行较大的程序，但本质不同。
段覆盖技术 是程序员在程序设计阶段人为安排各段的装入和覆盖关系，操作系统并不自动管理；
而 段式虚拟存储管理 是由操作系统自动根据程序执行情况动态调入和调出分段，对用户透明。
因此，段覆盖是用户控制的内存扩充技术，而段式虚拟存储管理是操作系统提供的自动虚拟化机制。

考前速背版

  • 所有段在辅存，部分段在主存
  • 用到哪段，调哪段
  • 不在内存 → 缺段中断
  • 段覆盖：用户控制
  • 段虚存：OS 自动控制

5.3 段页式存储管理

一句话理解

段页式就是“先分段、段内再分页”，既保留程序逻辑结构，又避免每段必须连续存放。

为什么要提出段页式

分段有优点：

  • 逻辑清晰
  • 易共享、易保护

但分段也有缺点：

  • 每段仍需连续空间
  • 有外碎片

分页有优点：

  • 离散分配
  • 无外碎片

于是自然想到把两者结合：段页式

基本思想

  1. 按程序逻辑结构先分段
  2. 每一段再划分为若干页面
  3. 主存按页框分配
  4. 段内各页可离散装入不同页框

这样：

  • 从用户角度仍有段
  • 从系统管理角度借助分页高效分配

地址结构

逻辑地址通常由三部分构成：

数据结构

段表项内容

  • 页表始址
  • 页表长度
  • 标志位

页表项内容

  • 页框号
  • 标志位

也就是说：

  • 每段对应一张页表
  • 段表先找到该段的页表
  • 页表再找到具体页框

段页式地址转换过程

Step 1：查段表

由段号找到该段的页表起始地址和页表长度。

Step 2：检查页号

判断页号是否超过该段页表长度。

Step 3：查页表

找到该页对应页框号。

Step 4：形成物理地址

将页框号与页内位移拼接。

段页式的优点

  1. 保留分段的逻辑意义
  2. 利用分页消除外碎片
  3. 支持段级共享和保护
  4. 适合大型复杂系统

段页式的缺点

  1. 地址转换复杂
  2. 需要段表 + 页表
  3. 管理开销较大
  4. 更依赖硬件支持（TLB、MMU 等）

段页式虚拟存储

课件也指出，段页式还可进一步扩展成虚拟存储：

  • 可只装入部分段
  • 或只装入段中的部分页面

这实际上是现代系统中比较强大的内存管理形态。

易错点

  1. 段页式不是“先分页后分段”，而是先分段再对每段分页。
  2. 段页式中用户看到的是段，系统内部用页框离散存放。
  3. 段页式虽然消除了外碎片，但仍可能有页内碎片。
  4. 地址转换是“段表一次 + 页表一次”，不是只查一次表。

题目：段页式存储管理结合了分段和分页的哪些优点？

答：
段页式存储管理先按程序逻辑结构分段，再对每一段进行分页。
它结合了分段和分页的优点：

  1. 保留分段的逻辑意义，便于共享、保护和模块化管理；
  2. 利用分页实现离散分配，使段不必整体连续存放，从而消除了外碎片；
  3. 适合扩展为虚拟存储管理，可以只装入部分段或段中的部分页面。
    其缺点是地址转换较复杂，需要段表和页表双重支持。

考前速背版

  • 先分段，段内再分页
  • 地址：段号 + 页号 + 页内位移
  • 查段表 → 查页表 → 得页框号
  • 优点：逻辑性 + 高利用率
  • 缺点：结构复杂

分页和分段的主要区别在于划分依据不同。

八、期末高频简答题模板

1. 存储管理的主要功能

答题模板：
存储管理的主要功能包括地址转换、主存空间的分配与回收、主存共享、存储保护以及主存扩充。其中地址转换负责把逻辑地址映射成物理地址；分配与回收负责管理进程装入和撤离时的内存使用；共享和保护保证多进程安全高效共存；主存扩充则通过对换和虚拟存储技术扩大可用内存空间。

2. 为什么分页能消除外碎片

答题模板：
分页把主存划分为大小固定的页框，把进程划分为同样大小的页面。页面可以离散装入任意空闲页框，不再要求整个进程连续存放，因此不会像可变分区那样因连续空间不足而产生外碎片。不过分页仍可能在进程最后一页产生少量内碎片。

3. 为什么虚拟存储器能工作

答题模板：
虚拟存储器能够工作的根本原因是程序执行具有局部性，包括时间局部性和空间局部性。程序在任一时刻只需要访问其中一部分代码和数据，因此操作系统可以只把当前所需部分装入主存，其余部分放在辅存中，按需动态调入，从而在较小主存上运行较大的程序。

4. 为什么需要快表 TLB

答题模板：
在页式存储管理中，若页表位于主存，则一次逻辑地址访问通常需要两次主存访问：一次查页表，一次访问目标数据。为提高地址转换速度，系统设置快表 TLB，将部分常用页表项保存在高速联想存储器中。若命中 TLB，就可减少一次主存访问，从而显著提升效率。

5. 分页与分段的区别

答题模板：
分页按固定大小划分地址空间，是系统为了提高内存利用率而采用的物理管理方式，用户不可见；分段按程序逻辑结构划分，是面向用户的逻辑管理方式，用户可见。分页页长固定、无外碎片但可能有内碎片；分段段长可变、便于共享和保护，但会产生外碎片。

九、全章易错点总汇

  1. 这章是操作系统存储管理，不是编译原理。
  2. 逻辑地址 != 物理地址。
  3. 静态重定位在装入时做，动态重定位在执行时做。
  4. 固定分区主要是内碎片。
  5. 可变分区主要是外碎片。
  6. 分页消除外碎片，但仍可能有内碎片。
  7. 分段按逻辑划分，分页按固定大小划分。
  8. TLB 不是 Cache，TLB 缓存页表项。
  9. 缺页中断/缺段中断不是错误，而是正常机制。
  10. 请求分页是按需调入，不是只调第一页。
  11. OPT 最优但不能真正精确实现。
  12. FIFO 可能出现 Belady 异常。
  13. LRU 利用时间局部性。
  14. 多级页表解决页表过大问题。
  15. 反置页表按页框组织，不按虚页组织。
  16. 段页式是先分段再分页。

十、考前速背版

1. 基础概念

  • 逻辑地址：程序使用的地址
  • 物理地址：内存真实地址
  • 重定位：逻辑地址 → 物理地址

2. 存储管理功能

  • 地址转换
  • 分配与回收
  • 共享
  • 保护
  • 扩充

口诀：转分共保扩

3. 连续分配

  • 单用户连续：最简单
  • 固定分区：多道，内碎片
  • 可变分区：灵活，外碎片
  • 可变分区算法：First / Next / Best / Worst

4. 虚拟存储器

  • 基础：局部性原理
  • 思想：部分装入，随用随调
  • 异常：缺页中断、缺段中断
  • 问题：抖动

5. 分页

  • 页：逻辑块
  • 页框：物理块
  • 页表：页 → 页框
  • 地址：页号 + 页内偏移
  • 公式：PA = 页框号 × 页大小 + 偏移
  • 优点：无外碎片
  • 缺点：页表开销、最后一页可能内碎片

6. TLB

  • 快表，缓存页表项
  • 命中快，未命中查主存页表
  • 用联想存储器

7. 页面替换

  • OPT：最优，理论算法
  • FIFO：简单，可能 Belady 异常
  • LRU：接近最优
  • Clock：近似 LRU

8. 分段

  • 地址：段号 + 段内位移
  • 公式：PA = 段基址 + 段内位移
  • 优点：逻辑清晰、共享方便、保护方便
  • 缺点：外碎片

9. 段页式

  • 先分段，段内再分页
  • 地址：段号 + 页号 + 页内位移
  • 查段表 → 查页表 → 得物理地址

10. 高频一句话

  • 分页面向系统，分段面向用户。
  • 分页提高利用率，分段提高逻辑性。
  • 虚拟存储器靠局部性。
  • 没有硬件支持，就没有高效现代内存管理。

结语式总复盘（可直接考前看）

如果你只剩最后几分钟复习，这章抓住下面主线就够了：

  1. 为什么要存储管理？
    因为多个进程要共享内存，程序地址要转换，内存还不够用。

  2. 最早怎么做？
    连续分配：单用户连续、固定分区、可变分区。
    问题分别是简单但效率低、内碎片、外碎片。

  3. 怎么优化？
    分页：把程序拆成固定大小页，把内存拆成页框，离散放置。
    好处是消除外碎片，适合虚拟存储。

  4. 分页最大问题是什么？
    地址转换慢，所以需要页表和 TLB。

  5. 内存不够怎么办？
    虚拟存储器：只装一部分，缺了再调。
    依据是程序局部性。

  6. 如果要更符合程序逻辑呢？
    用分段；如果既要逻辑性又要分页效率，就用段页式。

只要把这条主线讲顺，这章的大题和简答题就基本稳了。
